Cara Kerja Sistem — Gambaran Utuh

Offloading pemrosesan paket data seluler (GTP-U) dari CPU laptop ke chip khusus FPGA Genesys Virtex-5. Laptop cukup menampilkan dashboard; FPGA menangani banjir paket berkecepatan tinggi & mengirim balik statistik ringkas.

1 · Radio (USRP)

Tangkap sinyal seluler dari udara, laptop decode lapisan PHY jadi paket GTP-U mentah.

SUMBER SINYAL
→

2 · Host (ThinkPad X230)

Bungkus / teruskan paket ke FPGA lewat kabel LAN. Pilih 1 dari 3 metode ingest (halaman 2).

JEMBATAN
→

3 · FPGA Virtex-5

Bongkar tiap amplop paket di hardware, baca GTP-U, hitung statistik secara paralel & instan.

OTAK ANALISIS
→

4 · Dashboard Web

Laptop terima angka ringkas (~25 KB/s) via UDP, tampilkan monitoring real-time di browser.

Tiap paket mengalir lewat "pipa rangkaian" yang sama — karena fisik & paralel, jauh lebih cepat daripada CPU yang memproses berurutan.

OUTPUT Yang Tampil di Dashboard

  • 0x01 · Global Stats — total user aktif total_imsi=142, uplink 18000 pps · tiap 1 dtk
  • 0x02 · Per-TEID — status tiap sesi IMSI·QFI·ACTIVE
  • 0x03 · Event — notif real-time (mis. Create Session)
  • 0x04 · Protocol Dist — komposisi gtp_u·pfcp·bssgp

3 Cara Memasukkan Paket ke FPGA

"Ingest" = jalur paket dari laptop ke FPGA. Tiga opsi dengan trade-off kecepatan vs kerumitan. Strategi: mulai Metode 3 (buktikan konsep) → naik Metode 1 (produksi) → Metode 2 (kelas 5G).
